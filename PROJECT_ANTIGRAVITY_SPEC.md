This specification establishes the strict architectural, logical, and typographic boundary conditions for the **Vestira Concierge** engine. It bridges the visual prototypes created on the Google Stitch infinite canvas with the deterministic execution requirements of the local codebase using the **Antigravity Framework**. 

Every automated generation agent, code model, and engineer interfacing with this workspace via the Model Context Protocol (MCP) server must treat this document as an un-bypasable runtime rule framework.

## 🎨 2. Design System Tokens & Strict CSS/Tailwind Framework

To ensure the codebase perfectly implements the lookbook aesthetic generated by the Stitch canvas, all styling layers must strictly be constructed out of this localized Design Token system. No third-party random Tailwind primitives, arbitrary shadows, or vibrant primary colors are allowed.

### 🪙 Token Registry Matrix

| Design Token Primitive | System CSS Custom Property | Tailwind 4 Layer Equivalent | Purpose & Boundary Application |
| :--- | :--- | :--- | :--- |
| **Canvas Background** | `--bg-warm-linen` | `bg-[#FAF0E6]` | Primary workspace background surface across client views |
| **Primary Accent** | `--fg-obsidian-velvet` | `bg-[#09090B]` or `text-[#09090B]` | Main typography color, active buttons, solid focus frames |
| **Surface Element** | `--surface-white` | `bg-[#FFFFFF]` | Floating interaction cards, data rows, form consoles |
| **AI Interception Tint**| `--tint-champagne` | `bg-[#F2EBD9]` | Chat message bubbles, AI recommendations, layout updates |
| **Micro-Borders** | `--border-muted-zinc` | `border-[#E4E4E7]` | Clean structural boundary lines; strictly replaces drop-shadows |

### 📐 Geometry, Corner Radius, and Shadow Constraints
- **Container Boundary Corners:** Main display item blocks, modular sections, and parent structural elements must use exactly `rounded-xl` (16px).
- **Interactive Component Corners:** Standard input fields, toggle buttons, and text fields must use exactly `rounded-md` (12px).
- **Data Status Badges:** Inline data badges, vector process markers, and semantic indicators must use exactly `rounded-sm` (8px).
- **Shadow Limitation Policy:** Default Tailwind shadow layers (`shadow-lg`, `shadow-2xl`) are explicitly banned. Ambient occlusion must be handled entirely by fine 1px border frames (`border border-zinc-100` or `#E4E4E7`).

## 🖋️ 3. Typographic Scaling & Rules

Typographic rhythm defines the editorial presence of Vestira Concierge. Code must enforce absolute hierarchy by splitting families based on functional utility:

### 🎭 Font Role Allocation
1. **Editorial Headers (`font-serif`):** Implements *Playfair Display* or *Didot* configurations. Must always be styled with tight character spacing (`tracking-tight`) and light/medium weights. Use exclusively for page-level headers, major section headers, product names, and transaction totals.
2. **Operational UI (`font-sans`):** Implements *Inter* or *Helvetica Neue* styles. Always set with standard text tracking. Applied across form labels, buttons, search input values, database rows, table metadata metrics, and data numbers.

### 📊 Code Typography Mapping Template
```tsx
// Standard Editorial Heading Component Core Rule
export const EditorialHeading = ({ text }: { text: string }) => (
  <h1 className="font-serif text-4xl font-light tracking-tight text-[#09090B] sm:text-5xl">
    {text}
  </h1>
);

// Standard Functional Metric Code Rule
export const InterfaceMetric = ({ value }: { value: string }) => (
  <span className="font-sans text-sm font-medium tracking-normal text-[#09090B]">
    {value}
  </span>
);
```

## ⚙️ 4. Structural Layout Rules & Route Contracts

The Antigravity Framework implements three structural shell profiles to manage view scopes. Code templates must never blend features across layout domains.

### 🔳 Layout Alpha — Standard Customer Portal Shell
- **Routes:** `app/(customer)/**/*`
- **Shell Layout:** Top sticky translucent menu bar (`fixed top-0 w-full backdrop-blur-md bg-white/80 border-b border-[#E4E4E7] z-50`).
- **Layout Boundaries:** Main container components wrapped inside a rigid `max-w-7xl` or `max-w-5xl` block with safe horizontal spacing gutters.
- **AI Presence:** The persistent circular assistant trigger bubble floating permanently at `fixed bottom-6 right-6 z-40 bg-[#F2EBD9]`.

### 🔲 Layout Beta — Immersive AI Workspace Shell
- **Routes:** `app/(ai)/concierge/page.tsx`
- **Shell Layout:** Completely structural, forcing full layout height constraints (`h-screen w-screen overflow-hidden bg-[#FAF0E6]`). Customer headers, navigation tracks, and floating buttons are programmatically omitted.
- **Grid Splits:** Grid is strictly broken into an asymmetrical 35% fixed chat log column (left) and an independent 65% generative styling staging canvas workspace area (right).

### 📐 Layout Gamma — Seller Operational Workspace Shell
- **Routes:** `app/(seller)/**/*`
- **Shell Layout:** Structural operational view layout featuring a persistent 260px wide static left-side panel container (`w-[260px] h-screen fixed left-0 top-0 bg-white border-r border-[#E4E4E7]`). Customer layouts and consumer assistant popups are stripped out.
- **Main Workspace Canvas:** The remainder of the screen space utilizes `pl-[260px] bg-[#FAF0E6] min-h-screen` to display high-density management tables, analytical metrics, or asset pipeline processing fields.

## 💻 5. Coding Principles & Logic Rules

When converting active Stitch frames into executable Next.js 15 / React Server Component (RSC) states under the Antigravity architecture, code execution must comply with these programming rules:

### 🧩 Rule 5.1: Clean Component Partitioning
- Views must be split into **Pure Data Server Presentation Components** (handling catalog metrics, database queries, metadata schemas) and **Active Interactive Client Logic Slices** (handling toggles, file uploads, sliders).
- Use `"use client"` exclusively for explicit click handlers, form validation fields, sliders, custom dropdown views, and audio streams.

### 💾 Rule 5.2: Safe State Transitions (Stitch Mapping)
- Layout mutations (like opening the contextual chat drawer from the Product Detail Page) must use deterministic React state properties rather than generic document level element shifts.
- Any overlay element (like the sliding dynamic basket drawer or side contextual help box) must feature clean event hooks that handle backdrop blurring and quick click-outside closes.

### 🔍 Rule 5.3: Type Definition Enforcements
Every layout item, order vector data row, or style preference record must pass strict TypeScript definitions. No `any` type escapes are allowed in the runtime environment.
